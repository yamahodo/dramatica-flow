# 模拟优先的 AI 小说创作系统：架构构想

**初版**：2026-04-24
**修订**：2026-04-25（v2，含独立 review 修正）
**状态**：构想（未落地到代码）
**定位**："修正主流的前沿探索"——路线有学术同路人，但仍属新鲜赛道

---

## 0. 背景与动机

Dramatica-Flow 项目用 Dramatica 叙事理论驱动 AI 长篇写作。本构想更激进：**把"剧情演绎"与"撰写成文"彻底分层**——先让多 agent 模拟出故事，再按需提炼成文字。

> **启发来源**：Storytron（Chris Crawford）因"模拟与讲述混在一层"失败；AI Dungeon 的单 prompt 架构同样导致叙事崩坏；Narratology vs Ludology 学术共识认为两种逻辑不可互相归约；日本 Group SNE"跑团→Replay→Novel"产业模式验证分层可行。

---

## 0.5 参考同路人（必读：我们不是第一个）

**这个方向已有明确的学术与商业先行者。下表是差异化锚点，不是竞品列表。**

| 项目 | 发表/发布 | 与本构想的关系 |
|---|---|---|
| **StoryBox** (arXiv 2510.11618) | 2025-10 | **最接近**：character agents + Storyteller agent，稳定产出 1.2 万字连贯长文 |
| **StoryVerse** (FDG 2024) | 2024 | abstract acts + Act Director（类似命运变量） |
| **IBSEN** (ACL 2024) | 2024 | Director-Actor 架构。⚠ 其 Director 会主动 reschedule plot——违反本构想铁律 |
| **Agents' Room** (ICLR 2025) | 2025 | DeepMind 给出 planning agent + writing agent + scratchpad 分工范本 |
| **Fable Showrunner (SHOW-1)** | 2023 | 商业 demo：多 agent + South Park 剧集 |
| **AI Town** (a16z) | 2023 | Stanford Generative Agents 的社区复现，仅生成行为无成文层 |

**本构想相对同路人的差异化**：
1. 显式 **Drama Manager / 导演层**（StoryBox 和多数系统用户介入接口弱）
2. **多世界线分叉**（世界树管理）—— 几乎无人做
3. **Replay 作为 first-class 中间表示**—— 结构化 + 可审计，无人系统化实现

**成熟度判断**：
- 1 万字连贯长文 2025-10 刚被 StoryBox 证明可行
- **10 万字级别尚无任何公开系统证明可行**
- 商业侧**零款工具**采用"模拟+成文分层"架构（Sudowrite / NovelCrafter / NovelAI 全是"提纲+续写"路线）

## 1. 核心哲学

| 原则 | 内容 |
|---|---|
| **分层不混** | 演绎归演绎，成文归成文。一个 agent 不同时承担两职 |
| **作者是导演，不是编剧** | 干预走外部情境层，不直接重写角色台词或内心 |
| **间接操纵是诚实的** | 改情境 = 改 prompt = 事实上的间接 override；承认这一点，不包装成"涌现圣洁" |
| **意外优先保留** | 涌现出的超预设走向，默认保留而不是规训——但有边界（见 §8.2） |

> **Review 修订点**：原 "系统永不 override 角色决策" 和 "涌现不可降维" 措辞被删除，因为命运变量本质就是一种间接操纵（Apocalypse World 的 Baker 也从未宣称 MC "不操纵角色"）。保留的可操作约束是 **"不直接重写台词/内心"**。

> **启发来源**：fabula/sjužet（俄国形式主义）、histoire/récit（Genette）；水野良 2024 年 CBR 采访教训"预设世界观硬塞会压垮即兴"。

## 2. 整体架构（三层产物 + 一个控制者）

```
┌─ 静态层：世界规则、角色卡、设定（人工维护）
│
├─ 模拟层（跑团）
│   ├─ 1 个 GM agent（世界权威 / NPC / 环境裁定）
│   ├─ N 个角色 agent（独立、有记忆、有性格）
│   ↑ 导演（用户）通过 Drama Manager 接口介入
│
├─ Replay 层（中间表示 IR；可读性为可选加成）
│   └─ 结构化事件 + 对话 + 旁白 + 元信息标注
│
└─ Novel 层（文学成品）
    └─ Replay 翻译成散文
```

> **Review 修订点**：Replay 从"first-class 独立成品"降级为"中间表示（IR）"。日本 Replay 作为独立出版物的成功有两个前提（TRPG 亚文化 + 日本小众市场），**中文网文市场对 Replay 可读性零验证**。可读性作为"加成目标"而非"核心主张"。

## 3. 模拟层：角色 agent 的内核

### 3.1 角色卡（静态骨架）
身份、人生渴望（super-objective）、核心性格 3-5 条、关系网（-100~+100）、初始知识边界。

### 3.2 记忆流（短程有效，长程是已知风险）
- **事件流**：时间序事件存储
- **反思**：定期提炼散乱事件为"领悟"
- **检索**：决策时召回相关记忆

### 3.3 决策循环
感知场景 → 召回记忆 → 结合性格目标推理 → 输出对话/动作/内心。

### 3.4 两条可操作铁律
1. **不直接重写角色台词或内心** ——所有干预走情境层
2. **信息边界严格**——角色只知亲历/听说/推理之事

> **Review 修订点**：原"铁律 3 允许犯错"被删除——LLM 实际上很难稳定"犯错"而不极端化表演。改为 3.5 中的工程关注点。

### 3.5 已知工程风险（不可忽略）

| 风险 | 描述 | 可能对策 |
|---|---|---|
| **记忆退化** | Stanford Agents 验证的是 25 人 / 2 天 / 社交场景。百万字长篇中向量检索精度塌陷、reflection 漂移、人设偏移都是**开放问题** | 分层摘要 + 主动老化 + 章外人设重锚（参考 StoryBox） |
| **人设不稳定** | 长序列中角色性格趋向均值化（所有角色说话一个调） | DPO 训练（参考 Wayfarer 的做法）+ 每幕前重注入性格摘要 |
| **"犯错"表演失败** | LLM 要求犯错时要么回归最优解、要么极端化 | 采用 MBTI/九型等性格向量 + 情绪状态机约束，而非 prompt 里说"你有缺点" |

> **Review 修订点**：原文把 Stanford Generative Agents 包装成"解决了角色内核"，射程完全对不上。这一节明确标记为"启发边界 + 已知风险"。

> **启发来源**：
> - 3.1 → Stanislavski 表演方法论
> - 3.2 → Stanford Generative Agents (Park et al. 2023) 的 memory / reflection / retrieval——**仅在短程社交场景验证**
> - 3.2 长程对策 → StoryBox hierarchical windowed summarization
> - 3.5 人设稳定训练 → LatitudeGames Wayfarer-Large 的 DPO 连贯性训练
> - 信息边界 → Marie-Laure Ryan possible worlds

## 4. 导演层：Drama Manager

用户可用操作：
- **压力时钟**：挂倒计时钟（"魔族入侵 0/12"），到点触发世界事件
- **命运变量注入**：关键时刻往世界扔外部变量（让某人出现、让灾难降临）
- **Front 管理**：威胁前线集群

**诚实的自我描述**：命运变量是**间接操纵**——改情境会改角色的条件分布，事实上影响其决策。这是**有意为之**的干预通道，不是"完全自由涌现"。

**心智模型**：Prep situations, not plots。

> **启发来源**：Drama Manager 概念（Mateas & Stern Façade 2005，但 Façade 是高度预写 beat 池，不是纯涌现）；Mark Riedl 的 Automated Story Director；Fronts + Clocks（Vincent Baker Apocalypse World）；D&D DMG / CoC Keeper 手册。

## 5. 多世界线演绎

### 5.1 分叉原则
**命运变量分叉**（改情境），不是**决策分叉**（指定行为）：
- ✅ 分支 A：师父没出现 / B：师父出现 / C：信物泄露
- ❌ 分支 A：林尘接了 / B：林尘拒绝

### 5.2 三档切换

| 档 | 谁想变量 | 谁决定 | 适合 |
|---|---|---|---|
| 手动 | 你 | 你 | 关键章节 |
| 半自动 | 系统列候选 | 你挑 | 日常创作 |
| 全自动 | 系统 | 系统跑 N 条供挑选 | 素材期 |

### 5.3 世界树
- 稀疏分叉（只在戏剧关键点）
- 操作：设为正史 / 存档 / 废弃 / 摘取 / 回分叉点

### 5.4 摘取（cherry-pick）的边界（Review 新增）

跨分支摘取不是无条件操作：

| 摘取类型 | 适用条件 | 风险 |
|---|---|---|
| **直接摘** | 正史角色的心理状态能支撑该对白/动作 | 低 |
| **改写摘** | 需要轻微调整措辞使兼容正史心理 | 中——**明确承认改意思** |
| **禁止摘** | 正史角色根本不会说出这句话（内心状态完全不兼容） | 不可用 |

### 5.5 根本取舍
纯命运分叉的多样性受限于角色性格空间。性格注定拒绝的角色，无论多少命运变量都不会自然接受——加码命运 vs 接受"这不是他的故事"。

> **Review 修订点**：5.4 摘取边界是补充的。原文宣称"分支里的精彩对白可摘回正史"未考虑心理状态不兼容问题。

## 6. Replay 层：中间表示（IR）

三层信息：
- **对话/动作层**（角色 agent 输出）
- **旁白/场景层**（GM agent 输出）
- **元信息层**（内心独白、状态变化、时钟、分叉候选——系统自动汇总）

### 两重确定价值
1. **结构化数据**——下游成文层精准读取
2. **可审计**——每句话可追到"谁说的、基于哪条记忆、面对什么情境"，调试故事像调试程序

### 可选加成：可读成品
Replay 有机会作为独立可读品类销售，但此主张**必须经过中文读者可读性前测**才能纳入产品路线图。**不作为核心主张**。

> **Review 修订点**：从"三重价值"降为"两重确定 + 一重待验证"。

> **启发来源**：Group SNE Replay 格式（日本 TRPG 产业经验，非中文验证）。

## 7. 成文层：翻译家，不是编剧

### 三个动作
- **删**：元信息、重复、无聊过渡
- **留**：对话基本原封、关键动作与内心、场景张力
- **加**：感官细节、时间呼吸、**必要的心理补写（边界见下）**

### 三个控制旋钮
- POV（聚焦）：单人跟随 / 全知 / 轮转
- 详略（时长）：过渡戏 5:1 压缩，高潮戏 1:5 扩写
- 叙述距离：紧贴内心 vs 冷眼旁观

### "不改意思"的精确定义（Review 修订）

原表述"不改意思"太模糊。精确化为：

> **成文层不得改变角色 agent 在该幕的核心决策和情感坐标**。可以：重排句序、增加感官细节、补写与当幕情感一致的心理描写。不可以：让角色做出 Replay 里不存在的决定、或表达与 Replay 里记录的情感相反的心理活动。

### 必避陷阱
角色的"意外反应"不要改回原设——那是涌现出的故事真相（水野良 2024 教训）。

> **启发来源**：Gérard Genette 叙事话语五维；Robert McKee Story 场景价值转换。

## 8. 长篇节奏管理

### 8.1 四层结构（修订：骨架二选一）

```
全书层：核心命题、主角弧光终点、大伏笔
  ↓
卷级骨架：Save the Cat 15 节拍（每 10-15 万字一卷）
  ↓
压力时钟层（导演主要工作台）：3-5 个 Countdown Clock
  ↓
幕级涌现（角色 agent 自主）
```

> **Review 修订点**：原文把 Dramatica 和 Save the Cat 并列是矛盾的——两套理论相互批评。**选 Save the Cat**：节拍具体可验证、网文实践友好、与涌现的冲突点更清晰。Dramatica 的双层需求（外部目标+内在渴望）保留为**角色卡字段**，不作为长篇结构理论。

### 8.2 骨架-涌现冲突解决协议（Review 新增）

当涌现走向与卷级节拍冲突时（比如第 10 幕应该 All Is Lost，但主角 agent 自主选择了"洒脱回家种田"）：

| 冲突偏离 | 处理方式 |
|---|---|
| **≤ 2 幕延迟** | 自动容忍，节拍顺延 |
| **3-5 幕延迟** | 系统预警，作者介入：要么施加更强命运变量，要么改卷级骨架 |
| **> 5 幕或方向反转** | 冻结本幕，作者必须做结构决定：回溯分支 / 承认本卷目的地变了 |

**协议原则**：涌现有容忍区间但非无限。节拍被吞噬超过阈值就必须有人做决定，系统不替你决定。

### 8.3 三条自动警戒线
- **角色弧光停滞**：主角 10 幕无内在变化
- **伏笔超期**：某伏笔 30 幕未推进（需作者**手动标注伏笔**，系统无法自动识别）
- **时钟卡住**：某时钟 15 幕无推进

> **Review 修订点**：原文暗示系统可自动识别涌现出的伏笔——不可能，伏笔的本体论需要作者意图。改为"作者标注 + 系统追踪推进度"。

### 8.4 网文特化
成文层硬约束：每章末尾必须留悬念钩子。

## 9. 作者心理契约与读者期待（Review 新增章节）

### 9.1 作者的角色转换代价
本系统要求作者**从"创作者"切换到"导演"**——从"我赋予主角意志"变成"我观察角色自主行动"。这**放弃了网文作者创作的核心心理回报**（自我投射+掌控爽感）。

**未解决的问题**：谁会长期使用一个把自己从主体降为观察者的工具？

**可能的缓解**：
- 提供"强介入模式"作为逃生阀（允许作者在关键幕直接改写 Replay，标记为"作者手动幕"，但不影响其他涌现幕）
- 把"看涌现"重新定位为一种更高阶的作者乐趣（"我设计了一个会自己活的世界，比我能想到的更精彩"）——此定位需用户群前测

### 9.2 读者口味的反方向风险
网文读者喜欢**强目的性、强主题、强作者意志**（爽文）。涌现产物天然松散、去中心、主题漂移。

**风险**：系统产出的小说可能在专业编剧看来有深度，但在网文市场上**被弃读**。

**对策**：成文层必须有"网文化"步骤——压缩日常、强化冲突、在每章结尾强插情绪钩子。这一层是**市场适配**，不是文学优化。

## 10. 中文语料与风格适配（Review 新增章节）

### 问题
文档通篇理论栈（Genette、McKee、Stanislavski、Snyder）均来自西方。中文网文有自己的语感谱系——烽火戏诸侯的古典、猫腻的散文化、辰东的爽感节奏——**不在任何西方理论里**。

### 已知局限
- GPT/Claude 对"渡劫""金丹""道心"的训练分布远不如英文奇幻
- 直接生成的修仙文容易"翻译腔"

### 可能路径
1. **风格参考语料库**：收集 10-20 本头部网文，成文层每次生成引入 few-shot 风格样本
2. **LoRA/DPO 微调**：在专用网文语料上训练成文层模型
3. **人机协作校稿**：作者轻度改稿 = 风格回注

**此议题超出理论范畴，属于工程实施层。**

## 11. IP 归属（Review 新增短节）

如果创意大部分来自 agent 涌现，谁是作者？中国著作权法对 AI 参与作品的权属**尚无成熟判例**。商业化前需明确立场（作者作为 Drama Manager 和 Curator 的角色是否构成独创性贡献）。本文件不作结论，列为**阻塞性商业问题**。

## 14. 下一步建议（Review 修订）

**优先级 1（启动前必做）**：
- **读 StoryBox 论文**（arXiv 2510.11618）并若有代码尝试跑通 demo——他们已在 1.2 万字级别跑通我们的核心架构，是最强参考
- 读 Agents' Room（arXiv 2410.02603）——planning/writing 分工范式
- 读 IBSEN（ACL 2024）—— 但注意其 Director 会 override 角色，是反面案例

**优先级 2（原型前必做）**：
- 跑最小原型：3 个角色 agent + 1 GM agent + 1 场景，验证涌现是否产生有趣故事
- 中文 Replay 可读性小规模用户前测（20-50 人）

**优先级 3（架构决策前必做）**：
- 调研 Inworld AI、AI Town、Wayfarer 的已有工程基建，评估可复用部分
- 确定 Dramatica-Flow 改造 vs 新起项目

## 附：核心一句话

> 传统 AI 写作是"给 AI 一个提纲让它填空"；本构想是"搭一个有生命的故事世界，让它自己长出故事，然后派文学家去记录"——**但涌现-控制的张力、长程记忆退化、中文市场验证、作者心理契约**是四块硬骨头，必须诚实面对，不能用哲学措辞回避。
